Bộ các câu lệnh của git:

Trc hết git có 3 vùng:

git --version: để kiểm tra phiên bản git hiện tại

git init: đẻ khởi kho lưu trữ trong 1 thư mục, file nào đó

git status: kiểm tra trạng thái các file đã đc thay đổi trong dự án

git add ten.file: để chuyển file từ vùng working sang Staging HOẶC git add . (thêm toàn bộ)

git commit -m"Nội dung": chuyển file từ vùng Staging sang vùng Repository

git log: để xem lịch sử các commit

git show commit_id: để xem chi tiết 1 commit nào đó

git diff: để xem chi tiết sự thấy đổi (điều kiện là file đó phải đang trang vung working)

gitk: Mở dashboard xem cho trực quan hơn

git checkout -- ten_file: bỏ đi thay đổi của file, trở lại trạng thái trc đó (lưu ý: chỉ dùng cho file đang trong vùng Working)

git reset Head ten_file HOẶC git reset ten_file: để chuyển file từ vùng Staging về lại vùng Working

git reset --soft commit_id: chuyển trạng thái file đã commit về trạng thái trước lúc commit (Tức từ Repository về Staging)

git reset --mixed commit_id: để chuyển từ trạng thái đã commit về trước lúc chạy lệnh git add . (hay từ vùng Repository về vùng Working)

Toàn bộ câu lệnh về nhánh:

Trc hết cần hiểu sơ bộ về nhánh:

git branch: xem danh sách nhánh

git checkout -b ten_nhanh: tạo 1 nhánh mới và chuyển sang nhánh đó
